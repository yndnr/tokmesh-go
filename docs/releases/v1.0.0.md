# TokMesh-Go v1.0.0（P1 完成版）

> 发布日期：2025-11-30  
> 覆盖范围：SDD P1（R1–R6、R11–R14、R19）

## 概览

TokMesh-Go v1.0.0 标志着 P1「MVP + mTLS」阶段正式完结。本次发布交付了专用 Session/Token 存储、完整会话生命周期 API、PKI/mTLS 安全基线、持久化与资源管理、管理端授权策略以及可复用的部署文档。自此，TokMesh-Go 可以作为轻量 IAM/SSO 会话管理服务投入实际环境。

## 亮点

1. **会话模型与生命周期（R1–R4）**  
   - `internal/session/` 提供专用 Session/Token 数据结构、多维索引与 CRUD API。  
   - HTTP 业务接口涵盖创建、校验、续期、撤销，CLI 同步支持管理操作。

2. **安全与部署（R5–R6、R13、R19）**  
   - 业务/管理双端口 + 强制 mTLS，支持证书策略与撤销名单。  
   - CLI 现可通过 `--api-key` / profile 存储自动附带 `X-API-Key` 头，方便在启用业务 API Key 的场景下使用。  
   - 新增速率限制、TLS1.2+、Cipher Suite 固化及错误信息隐藏等加固措施。

3. **持久化与资源管理（R11、R12、R24）**  
   - WAL/快照路径默认启用 AES-GCM 加密（`TOKMESH_ENCRYPTION_KEY`），可在没有密钥时退回明文模式。  
   - 内存阈值与 TTL 清理可在 CLI/管理 API 中实时触发与观测。

4. **开发者体验**  
   - `README.md` 更新了 API Key、速率限制、撤销列表与持久化加密配置说明。  
   - 新增 `golang.org/x/time/rate` 依赖以支持服务端限流；CLI profile 文件（`~/.tokmesh/cli.yaml`）可保存证书、CA 与 API Key。

## 升级指南

1. **依赖**：Go 1.24 以上（go.mod 已更新）。  
2. **配置**：  
   - 若启用 API Key，请设置 `TOKMESH_BUSINESS_API_KEYS` 并更新 CLI profile。  
   - 若需加密 WAL/快照，请设置 64 位 hex 的 `TOKMESH_ENCRYPTION_KEY`。  
   - 证书撤销可通过 `TOKMESH_ADMIN_REVOKED_FINGERPRINTS` 配置 SHA256 指纹。  
3. **CLI**：升级到本版本后，运行 `tokmesh-cli cert install --api-key ...` 以保存新的 profile 参数。
