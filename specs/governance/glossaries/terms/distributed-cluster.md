# 分布式集群 (Distributed Cluster)

## 解释
分布式集群是指一组通过网络连接、协同工作的计算机节点。这些节点在逻辑上对外表现为一个单一的系统。集群通过冗余和负载均衡机制，提供比单机系统更高的可用性、可扩展性和容错能力。

## 使用场景
- **高可用服务**: 当某个节点发生故障时，其他节点接管工作，确保服务不中断。
- **大规模计算与存储**: 当单机资源无法满足需求时，通过增加节点水平扩展性能。
- **状态同步**: 需要在地理位置分散的多个节点间保持数据一致性的场景。

## 示例
### TokMesh 节点集群
假设 TokMesh 部署了 3 个节点（Node A, Node B, Node C）组成一个集群：
1.  **Node A** 接收到客户端的写请求。
2.  **Node A** 通过共识算法（如 Raft）将数据复制给 **Node B** 和 **Node C**。
3.  一旦大多数节点（如 2 个）确认写入，集群即向客户端返回成功。
4.  如果 Node A 宕机，Node B 和 Node C 会自动选举出新的 Leader 继续提供服务。
