# 想法: API Key 鉴权体系

状态: 已批准

## 原始想法

为了保障 TokMesh 服务的安全性，需建立一套可管理的 **API Key** 鉴权体系，作为所有接口调用的基础凭证。

### 核心功能
1.  **多维度权限控制**: API Key 不仅仅是一个字符串，它携带权限元数据。
    *   **Admin 角色**: 拥有所有权限，包括创建/吊销其他 Key、管理集群节点、执行备份还原等。
    *   **User 角色**: 仅拥有业务层面的读写权限（如验证 Token、创建 Session）。
2.  **生命周期管理**: 支持 Key 的创建、启用、禁用、轮转（Rotate）和删除。
3.  **鉴权位置**: 无论是 HTTP 接口还是 Redis 兼容接口，都必须在入口处校验 API Key。
