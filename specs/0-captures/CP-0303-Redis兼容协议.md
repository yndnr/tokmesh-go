# 想法: Redis 兼容协议

状态: 已批准

## 原始想法

提供一套兼容 **Redis 指令集** 的调用接口。

### 核心目的
*   **兼容老系统**: 允许遗留系统在不修改代码（或仅修改配置）的情况下，将 TokMesh 作为 Session 存储后端接入。
*   **降低迁移成本**: 利用现有的 Redis 客户端生态。

### 设计限制
*   **非全量实现**: **不完全实现** Redis 的所有功能（如 List, Set, ZSet 等复杂数据结构如果不相关则不实现）。
*   **仅针对会话**: 仅实现与会话管理强相关的指令（如 `GET`, `SET`, `DEL`, `EXPIRE`, `TTL`）。
*   **令牌校验**: 对于 Redis 协议中不存在的"令牌校验"逻辑，**需要评估**是否通过自定义扩展指令（如 `TM.VALIDATE`）提供，或者建议用户转向 OpenAPI 接口。
*   **鉴权**: 需通过 Redis 的 `AUTH` 指令适配 API Key 鉴权。
