# RQ-254904 - 协议适配层

状态: 草稿
优先级: P0（必须）
创建日期: 2025-12-06
来源文档: CP-254901
依赖需求: RQ-254902, RQ-254903

## 需求来源

TokMesh 需要支持多种访问协议，以满足不同场景的需求：
- **gRPC**：高性能首选，适合内部微服务调用
- **REST**：通用兼容，适合 Web 应用和第三方集成
- **Redis 协议**：便于从 Redis 迁移，降低接入成本

## 需求描述

### 1. gRPC 协议

#### 1.1 服务定义

**业务服务（Business Service）**：
```protobuf
service BusinessService {
  // 会话操作
  rpc CreateSession(CreateSessionRequest) returns (CreateSessionResponse);
  rpc GetSession(GetSessionRequest) returns (GetSessionResponse);
  rpc ValidateSession(ValidateSessionRequest) returns (ValidateSessionResponse);
  rpc RefreshSession(RefreshSessionRequest) returns (RefreshSessionResponse);
  rpc DeleteSession(DeleteSessionRequest) returns (DeleteSessionResponse);

  // 令牌操作
  rpc StoreToken(StoreTokenRequest) returns (StoreTokenResponse);
  rpc ValidateToken(ValidateTokenRequest) returns (ValidateTokenResponse);
  rpc RevokeToken(RevokeTokenRequest) returns (RevokeTokenResponse);

  // 批量操作
  rpc BatchValidateSessions(BatchValidateSessionsRequest) returns (BatchValidateSessionsResponse);
  rpc BatchValidateTokens(BatchValidateTokensRequest) returns (BatchValidateTokensResponse);
  rpc BatchDeleteSessions(BatchDeleteSessionsRequest) returns (BatchDeleteSessionsResponse);
}
```

**管理服务（Admin Service）**：
```protobuf
service AdminService {
  // 会话管理
  rpc ListSessions(ListSessionsRequest) returns (ListSessionsResponse);
  rpc GetSessionDetail(GetSessionDetailRequest) returns (GetSessionDetailResponse);
  rpc KickSession(KickSessionRequest) returns (KickSessionResponse);
  rpc BatchKickSessions(BatchKickSessionsRequest) returns (BatchKickSessionsResponse);

  // 令牌管理
  rpc ListTokens(ListTokensRequest) returns (ListTokensResponse);
  rpc RevokeTokenAdmin(RevokeTokenAdminRequest) returns (RevokeTokenAdminResponse);
  rpc BatchRevokeTokens(BatchRevokeTokensRequest) returns (BatchRevokeTokensResponse);

  // 统计
  rpc GetOnlineStats(GetOnlineStatsRequest) returns (GetOnlineStatsResponse);
  rpc GetCapacityStats(GetCapacityStatsRequest) returns (GetCapacityStatsResponse);
  rpc GetPerformanceStats(GetPerformanceStatsRequest) returns (GetPerformanceStatsResponse);

  // 策略配置
  rpc GetTTLPolicy(GetTTLPolicyRequest) returns (GetTTLPolicyResponse);
  rpc UpdateTTLPolicy(UpdateTTLPolicyRequest) returns (UpdateTTLPolicyResponse);
  rpc GetRateLimitPolicy(GetRateLimitPolicyRequest) returns (GetRateLimitPolicyResponse);
  rpc UpdateRateLimitPolicy(UpdateRateLimitPolicyRequest) returns (UpdateRateLimitPolicyResponse);
}
```

#### 1.2 gRPC 特性支持

| 特性 | 支持 | 说明 |
|------|------|------|
| Unary RPC | 是 | 所有接口使用 Unary 模式 |
| Server Streaming | 否 | MVP 不支持 |
| Client Streaming | 否 | MVP 不支持 |
| Bidirectional Streaming | 否 | MVP 不支持 |
| Deadline/Timeout | 是 | 客户端可设置超时 |
| Metadata | 是 | 用于传递认证信息 |
| Interceptors | 是 | 用于认证、日志、限流 |

#### 1.3 gRPC 端口配置

```yaml
grpc:
  business:
    address: 0.0.0.0:50051
    tls:
      enabled: true
      cert_file: /etc/tls/server.crt
      key_file: /etc/tls/server.key
  admin:
    address: 127.0.0.1:50052  # 仅内网访问
    tls:
      enabled: true
      mode: mtls  # 管理接口使用 mTLS
      cert_file: /etc/tls/server.crt
      key_file: /etc/tls/server.key
      ca_file: /etc/tls/ca.crt
```

### 3. Redis 协议

#### 3.1 支持的命令（最小子集）

**基础命令**：

| 命令 | 语法 | 说明 |
|------|------|------|
| SET | SET key value [EX seconds] | 设置键值，支持过期时间 |
| GET | GET key | 获取键值 |
| DEL | DEL key [key ...] | 删除键 |
| EXISTS | EXISTS key [key ...] | 检查键是否存在 |
| EXPIRE | EXPIRE key seconds | 设置过期时间 |
| TTL | TTL key | 获取剩余过期时间 |
| PTTL | PTTL key | 获取剩余过期时间（毫秒） |
| PING | PING [message] | 连接测试 |
| QUIT | QUIT | 关闭连接 |

#### 3.2 键值映射

为了兼容现有 Redis 使用方式，提供以下映射：

**会话键**：
- 键格式：`session:{session_id}` 或 `sess:{session_id}`
- 值格式：JSON 字符串（会话对象）

**令牌键**：
- 键格式：`token:{token_id}` 或 `tok:{token_id}`
- 值格式：JSON 字符串（令牌对象）

**示例**：
```redis
SET session:abc123 '{"user_id":"12345","device_type":"web"}' EX 3600
GET session:abc123
DEL session:abc123
TTL session:abc123
```

#### 3.3 限制说明

- 仅支持基础命令，不支持 Lua 脚本
- 不支持事务（MULTI/EXEC）
- 不支持发布订阅（PUB/SUB）
- 不支持集群模式命令
- 键必须符合 `session:` 或 `token:` 前缀格式

#### 3.4 Redis 端口配置

```yaml
redis:
  enabled: true
  address: 0.0.0.0:6379
  # 可选：密码认证
  password: ""
  # 可选：TLS
  tls:
    enabled: false
```

### 4. 协议通用要求

#### 4.1 TLS 支持

所有协议都支持 TLS：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 无 TLS | 明文传输 | 仅开发环境 |
| TLS | 服务端证书 | 业务 API |
| mTLS | 双向证书认证 | 管理 API |

#### 4.2 认证方式

**业务 API**：
- API Key（通过 Header 或 Query Parameter）
- mTLS 客户端证书（可选）

**管理 API**：
- API Key + mTLS（推荐）
- 仅 mTLS（高安全场景）

**认证头格式**：
```
Authorization: Bearer <api_key>
X-API-Key: <api_key>
```

#### 4.3 健康检查端点

所有协议提供健康检查：

| 协议 | 端点 | 说明 |
|------|------|------|
| gRPC | grpc.health.v1.Health | 标准 gRPC 健康检查 |
| REST | GET /health/live | 存活探针 |
| REST | GET /health/ready | 就绪探针 |
| Redis | PING | 返回 PONG |

### 5. 错误码映射

#### 5.1 gRPC Status Code 映射

TokMesh 内部错误码到 gRPC Status Code 的映射关系：

| TokMesh 错误码 | gRPC Status Code | 说明 |
|---------------|------------------|------|
| OK | OK (0) | 成功 |
| INVALID_ARGUMENT | INVALID_ARGUMENT (3) | 参数错误（字段缺失、格式错误、长度超限） |
| NOT_FOUND | NOT_FOUND (5) | 资源不存在（会话/令牌/API Key 不存在） |
| ALREADY_EXISTS | ALREADY_EXISTS (6) | 资源已存在（重复创建） |
| EXPIRED | FAILED_PRECONDITION (9) | 资源已过期 |
| REVOKED | FAILED_PRECONDITION (9) | 资源已吊销 |
| UNAUTHENTICATED | UNAUTHENTICATED (16) | 未认证（缺少/无效的认证凭据） |
| PERMISSION_DENIED | PERMISSION_DENIED (7) | 权限不足（角色权限不满足） |
| RATE_LIMITED | RESOURCE_EXHAUSTED (8) | 触发限流 |
| RESOURCE_EXHAUSTED | RESOURCE_EXHAUSTED (8) | 资源耗尽（内存/会话数达上限） |
| QUERY_TOO_BROAD | INVALID_ARGUMENT (3) | 查询范围过大 |
| OPERATION_TOO_FREQUENT | RESOURCE_EXHAUSTED (8) | 操作过于频繁 |
| INTERNAL_ERROR | INTERNAL (13) | 内部错误 |
| UNAVAILABLE | UNAVAILABLE (14) | 服务不可用 |
| DEADLINE_EXCEEDED | DEADLINE_EXCEEDED (4) | 请求超时 |

**gRPC 错误响应格式**：

```protobuf
// 使用 google.rpc.Status
message Status {
  int32 code = 1;                    // gRPC status code
  string message = 2;                // 错误描述
  repeated google.protobuf.Any details = 3;  // 错误详情
}

// 错误详情使用 ErrorInfo
message ErrorInfo {
  string reason = 1;                 // TokMesh 错误码（如 "INVALID_ARGUMENT"）
  string domain = 2;                 // "tokmesh.io"
  map<string, string> metadata = 3;  // 附加信息（如 field: "session_id"）
}
```

**示例**：

```json
{
  "code": 3,
  "message": "session_id is required",
  "details": [{
    "@type": "type.googleapis.com/google.rpc.ErrorInfo",
    "reason": "INVALID_ARGUMENT",
    "domain": "tokmesh.io",
    "metadata": {
      "field": "session_id"
    }
  }]
}
```

#### 5.2 HTTP 状态码映射

TokMesh 内部错误码到 HTTP 状态码的映射关系：

| TokMesh 错误码 | HTTP Status | 说明 |
|---------------|-------------|------|
| OK | 200 OK | 成功 |
| INVALID_ARGUMENT | 400 Bad Request | 参数错误 |
| NOT_FOUND | 404 Not Found | 资源不存在 |
| ALREADY_EXISTS | 409 Conflict | 资源已存在 |
| EXPIRED | 410 Gone | 资源已过期 |
| REVOKED | 410 Gone | 资源已吊销 |
| UNAUTHENTICATED | 401 Unauthorized | 未认证 |
| PERMISSION_DENIED | 403 Forbidden | 权限不足 |
| RATE_LIMITED | 429 Too Many Requests | 触发限流 |
| RESOURCE_EXHAUSTED | 503 Service Unavailable | 资源耗尽 |
| QUERY_TOO_BROAD | 400 Bad Request | 查询范围过大 |
| OPERATION_TOO_FREQUENT | 429 Too Many Requests | 操作过于频繁 |
| INTERNAL_ERROR | 500 Internal Server Error | 内部错误 |
| UNAVAILABLE | 503 Service Unavailable | 服务不可用 |
| DEADLINE_EXCEEDED | 504 Gateway Timeout | 请求超时 |

**HTTP 错误响应格式**：

```json
{
  "error": {
    "code": "INVALID_ARGUMENT",
    "message": "session_id is required",
    "details": {
      "field": "session_id",
      "constraint": "required"
    }
  }
}
```

**HTTP 响应头**：

| 场景 | 响应头 | 说明 |
|------|--------|------|
| 限流 | `Retry-After: 60` | 建议重试时间（秒） |
| 限流 | `X-RateLimit-Limit: 100` | 限流上限 |
| 限流 | `X-RateLimit-Remaining: 0` | 剩余配额 |
| 限流 | `X-RateLimit-Reset: 1701864000` | 配额重置时间（Unix 时间戳） |
| 认证 | `WWW-Authenticate: Bearer` | 认证方式提示 |

#### 5.3 Redis 协议错误映射

Redis 协议使用简单的错误响应格式：

| TokMesh 错误码 | Redis 错误前缀 | 示例响应 |
|---------------|---------------|---------|
| INVALID_ARGUMENT | ERR | `-ERR invalid argument: key format must be session:{id} or token:{id}` |
| NOT_FOUND | - | `$-1\r\n`（nil bulk string，GET 操作）<br>`:0\r\n`（整数 0，EXISTS 操作） |
| UNAUTHENTICATED | NOAUTH | `-NOAUTH Authentication required` |
| PERMISSION_DENIED | NOPERM | `-NOPERM no permission to access this key` |
| RATE_LIMITED | BUSY | `-BUSY rate limit exceeded, retry after 60 seconds` |
| RESOURCE_EXHAUSTED | OOM | `-OOM command not allowed when used memory > maxmemory` |
| INTERNAL_ERROR | ERR | `-ERR internal error` |

**Redis 命令特殊处理**：

| 命令 | 键不存在时 | 键已过期时 |
|------|-----------|-----------|
| GET | 返回 nil | 返回 nil（并删除） |
| SET | 创建新键 | 覆盖（并重置 TTL） |
| DEL | 返回 0 | 返回 1（并删除） |
| EXISTS | 返回 0 | 返回 0（并删除） |
| TTL | 返回 -2 | 返回 -2（并删除） |
| EXPIRE | 返回 0 | 返回 0 |

#### 5.4 错误码详细说明

##### 参数错误（INVALID_ARGUMENT）

触发条件：
- 必需字段缺失
- 字段类型错误
- 字段值超出范围
- 字段长度超限（参考 RQ-254901 字段长度限制）
- 枚举值无效

```json
{
  "error": {
    "code": "INVALID_ARGUMENT",
    "message": "user_agent exceeds maximum length of 2048 characters",
    "details": {
      "field": "user_agent",
      "constraint": "max_length",
      "limit": 2048,
      "actual": 3000
    }
  }
}
```

##### 资源状态错误（EXPIRED / REVOKED）

触发条件：
- 会话/令牌已过期
- 会话/令牌已被吊销

```json
{
  "error": {
    "code": "EXPIRED",
    "message": "session has expired",
    "details": {
      "resource_type": "session",
      "resource_id": "sess_abc123",
      "expired_at": "2025-12-06T10:00:00Z"
    }
  }
}
```

##### 认证错误（UNAUTHENTICATED）

触发条件：
- 缺少 Authorization 头
- API Key 无效
- API Key 已吊销或过期
- mTLS 证书无效

```json
{
  "error": {
    "code": "UNAUTHENTICATED",
    "message": "invalid or expired API key",
    "details": {
      "auth_method": "api_key"
    }
  }
}
```

##### 限流错误（RATE_LIMITED）

触发条件：
- 请求频率超过限制

```json
{
  "error": {
    "code": "RATE_LIMITED",
    "message": "rate limit exceeded for kick operations",
    "details": {
      "limit": "10/minute",
      "retry_after": 45
    }
  }
}
```
