## 优先级说明

- P0（必须）：MVP阶段必须完成，是产品的核心功能
- P1（重要）：MVP阶段重要功能，可在MVP后补充完善
- P2（期望）：锦上添花的功能，根据资源情况安排

## MVP阶段核心需求

以下需求必须在MVP阶段完成：
1. R001 - [核心功能1]
2. R002 - [核心功能2]

## 文档命名规范

- 编号格式：R001, R002, R003...
- 文件命名：R<编号>-<标题>.md，其中<标题>与文档主标题保持一致
- 示例：R001-用户认证功能需求.md

## 文档结构

每个需求文档应包含：
1. 需求来源：客户需求/市场分析/技术驱动
2. 需求描述：清晰的功能或非功能需求说明
3. 验收标准：如何验证需求已满足
4. 优先级：P0（必须）/P1（重要）/P2（期望）
5. 相关设计：链接到对应的D系列文档
6. 状态：草稿/已批准/实施中/已完成/已废弃
