# RQ-254903 - 管理 API

状态: 草稿
优先级: P0（必须）
创建日期: 2025-12-06
来源文档: CP-254901, CP-254902, CP-254903
依赖需求: RQ-254901

## 需求来源

管理 API 是 TokMesh 面向安全运营和 IAM 管理员的接口，提供会话管理、多维度检索、统计分析和动态策略配置能力。这是 TokMesh 相比 Redis 的核心差异化能力之一。

## 需求描述

### 1. 设计原则

- **安全优先**：严格认证、完整审计、防止滥用
- **功能完整**：满足安全运营的日常管理需求
- **查询受控**：多维度检索但复杂度可预测
- **审计透明**：所有操作留痕可追溯

### 2. 会话管理

#### 2.1 查询会话列表

**功能**：按条件查询会话列表

**输入参数**：
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| user_id | string | 否 | 按用户 ID 筛选 |
| ip | string | 否 | 按 IP 地址筛选（精确匹配） |
| ip_prefix | string | 否 | 按 IP 网段筛选（CIDR，如 192.168.1.0/24） |
| device_type | enum | 否 | 按终端类型筛选 |
| session_type | enum | 否 | 按会话类型筛选 |
| status | enum | 否 | 按状态筛选（active/expired/revoked） |
| created_after | timestamp | 否 | 创建时间起始 |
| created_before | timestamp | 否 | 创建时间截止 |
| last_active_after | timestamp | 否 | 最后活跃时间起始 |
| last_active_before | timestamp | 否 | 最后活跃时间截止 |
| page_size | int | 否 | 每页数量，默认 20，最大 100 |
| page_token | string | 否 | 分页 Token |

**查询限制**：
- 最多支持 3 个维度组合查询
- 必须至少提供一个筛选条件（防止全表扫描）
- 时间范围不超过 7 天

**输出**：
| 字段 | 类型 | 说明 |
|------|------|------|
| sessions | []Session | 会话列表 |
| total_count | int | 总数（可选，大数据量时不返回） |
| next_page_token | string | 下一页 Token |

#### 2.2 获取会话详情

**功能**：获取单个会话的完整信息

**输入参数**：
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| session_id | string | 是 | 会话 ID |

**输出**：完整的会话对象 + 关联的令牌列表

#### 2.3 踢出会话（强制下线）

**功能**：强制终止指定会话

**输入参数**：
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| session_id | string | 是 | 会话 ID |
| reason | string | 否 | 踢出原因（记录到审计日志） |

**输出**：
| 字段 | 类型 | 说明 |
|------|------|------|
| success | bool | 操作是否成功 |
| revoked_tokens | int | 同时吊销的令牌数量 |

**审计日志**：记录操作者、被踢会话、原因、时间

#### 2.4 批量踢出会话

**功能**：按条件批量踢出会话

**输入参数**：
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| user_id | string | 二选一 | 踢出该用户的所有会话 |
| ip_prefix | string | 二选一 | 踢出该 IP 段的所有会话 |
| reason | string | 否 | 踢出原因 |
| exclude_session_ids | []string | 否 | 排除的会话（如保留当前会话） |

**输出**：
| 字段 | 类型 | 说明 |
|------|------|------|
| revoked_count | int | 踢出的会话数量 |
| revoked_tokens | int | 吊销的令牌数量 |

**限制**：单次最多踢出 1000 个会话

### 3. 令牌管理

#### 3.1 查询令牌列表

**功能**：按条件查询令牌列表

**输入参数**：
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| user_id | string | 否 | 按用户 ID 筛选 |
| session_id | string | 否 | 按关联会话筛选 |
| token_type | enum | 否 | 按令牌类型筛选 |
| status | enum | 否 | 按状态筛选 |
| issued_after | timestamp | 否 | 签发时间起始 |
| issued_before | timestamp | 否 | 签发时间截止 |
| page_size | int | 否 | 每页数量 |
| page_token | string | 否 | 分页 Token |

**输出**：令牌列表 + 分页信息

#### 3.2 吊销令牌

**功能**：吊销指定令牌

**输入参数**：
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| token_id | string | 是 | 令牌 ID |
| reason | string | 否 | 吊销原因 |

**审计日志**：记录操作者、被吊销令牌、原因、时间

#### 3.3 批量吊销令牌

**功能**：按条件批量吊销令牌

**输入参数**：
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| user_id | string | 二选一 | 吊销该用户的所有令牌 |
| session_id | string | 二选一 | 吊销该会话的所有令牌 |
| token_type | enum | 否 | 仅吊销指定类型 |
| reason | string | 否 | 吊销原因 |

### 4. 统计与分析

#### 4.1 在线统计

**功能**：获取当前在线统计

**输入参数**：
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| group_by | enum | 否 | 分组维度（device_type/session_type/ip_prefix） |

**输出**：
```json
{
  "total_sessions": 54321,
  "total_users": 12345,
  "breakdown": {
    "web": 30000,
    "mobile": 20000,
    "api": 4321

```

**功能**：获取存储容量统计

**输出**：

```json
  "sessions": {
    "count": 54321,
    "memory_bytes": 54321000

  "tokens": {
    "count": 108642,
    "memory_bytes": 21728400
  },
  "indexes": {
    "memory_bytes": 10000000

  "total_memory_bytes": 86049400

```

**功能**：获取性能指标

**输出**：
```json

  "qps": {

    "peak_1h": 50000
```

### 5. 策略配置

#### 5.1 查询 TTL 策略

**功能**：查询当前 TTL 策略配置

**输出**：
```json
{
  "session": {
    "normal": {"ttl": "1h", "sliding": true, "max_ttl": "24h"},
    "vip": {"ttl": "2h", "sliding": true, "max_ttl": "48h"},
    "admin": {"ttl": "30m", "sliding": false}
  },
  "token": {
    "access": {"ttl": "5m"},
    "refresh": {"ttl": "7d"},
    "admin": {"ttl": "1m"}
  }
}
```

#### 5.2 更新 TTL 策略

**功能**：动态更新 TTL 策略（热生效）

**输入参数**：
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| target | string | 是 | 策略目标（如 session.normal） |
| ttl | duration | 否 | 新的 TTL 值 |
| sliding | bool | 否 | 是否启用滑动 TTL |
| max_ttl | duration | 否 | 最大 TTL（仅 sliding=true 时有效） |

**审计日志**：记录操作者、变更内容、时间

#### 5.3 查询限流策略

**功能**：查询当前限流策略

#### 5.4 更新限流策略

**功能**：动态更新限流策略（热生效）

### 6. 安全要求

#### 6.1 认证

- 所有管理 API 必须经过认证
- 支持 API Key 认证
- 支持 mTLS 客户端证书认证
- 推荐使用 mTLS（安全性更高）

#### 6.2 授权

- 支持基于角色的访问控制（RBAC）
- 预定义角色：
  - `viewer`：只读权限（查询、统计）
  - `operator`：操作权限（踢人、吊销）
  - `admin`：管理权限（策略配置）

**角色权限矩阵**：

| 操作类别 | 具体操作 | viewer | operator | admin |
|---------|---------|--------|----------|-------|
| **会话查询** | 查询会话列表 | ✅ | ✅ | ✅ |
| | 获取会话详情 | ✅ | ✅ | ✅ |
| **会话管理** | 踢出会话 | ❌ | ✅ | ✅ |
| | 批量踢出会话 | ❌ | ✅ | ✅ |
| **令牌查询** | 查询令牌列表 | ✅ | ✅ | ✅ |
| **令牌管理** | 吊销令牌 | ❌ | ✅ | ✅ |
| | 批量吊销令牌 | ❌ | ✅ | ✅ |
| **统计分析** | 在线统计 | ✅ | ✅ | ✅ |
| | 容量统计 | ✅ | ✅ | ✅ |
| | 性能统计 | ✅ | ✅ | ✅ |
| **策略配置** | 查询 TTL 策略 | ✅ | ✅ | ✅ |
| | 更新 TTL 策略 | ❌ | ❌ | ✅ |
| | 查询限流策略 | ✅ | ✅ | ✅ |
| | 更新限流策略 | ❌ | ❌ | ✅ |

**权限设计原则**：
- **最小权限**：默认无权限，按需授予
- **职责分离**：查看、操作、配置三层分离
- **可扩展性**：支持自定义角色（设计阶段定义）

#### 6.3 限流

管理 API 采用**严格限流**策略：

| 操作类型 | 限流规则 |
|---------|---------|
| 查询操作 | 100 次/分钟 |
| 踢人/吊销 | 10 次/分钟 |
| 批量操作 | 1 次/分钟 |
| 策略变更 | 2 次/分钟 |

#### 6.4 审计

所有写操作必须记录审计日志：
- 操作者身份（API Key / 证书 DN）
- 操作类型
- 操作目标
- 操作参数
- 操作结果
- 时间戳
- 客户端 IP

### 7. 错误处理

扩展 RQ-254902 的错误码，增加：

| 错误码 | 说明 |
|--------|------|
| UNAUTHENTICATED | 未认证 |
| PERMISSION_DENIED | 权限不足 |
| QUERY_TOO_BROAD | 查询范围过大（需增加筛选条件） |
| OPERATION_TOO_FREQUENT | 操作过于频繁（触发限流） |

## 验收标准

### 功能验收

1. [ ] 会话列表查询支持多维度筛选
2. [ ] 会话列表查询支持分页
3. [ ] 踢出会话操作正常
4. [ ] 批量踢出操作正常
5. [ ] 令牌查询和吊销操作正常
6. [ ] 统计接口返回正确数据
7. [ ] 策略配置支持热生效

### 安全验收

1. [ ] 所有接口要求认证
2. [ ] RBAC 权限控制生效
3. [ ] 限流策略生效
4. [ ] 审计日志完整记录

### 性能验收

1. [ ] 单维度查询 P99 < 5ms
2. [ ] 多维度查询 P99 < 10ms
3. [ ] 统计查询 P99 < 20ms（可缓存）
