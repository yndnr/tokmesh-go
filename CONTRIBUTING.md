# 贡献指南

欢迎为 TokMesh-Go 提交代码、文档和设计改进。本项目采用 SDD 驱动开发流程，建议阅读以下文档后再开始工作：

- 需求与设计入口：`docs/sdd/README.md`
- v1 需求真相源：`docs/sdd/TokMesh-v1-02-requirements.md`
- 仓库级规范：`AGENTS.md`

## 基本流程

- 新功能或较大改动：  
  - 优先在 `docs/sdd/TokMesh-v1-04-backlog-inbox.md` 或相关 SDD 中补充/更新设计；  
  - 再按照实现蓝图（如 `docs/sdd/TokMesh-v1-30-P1-blueprint.md`）落地代码。

- 小改动 / Bug 修复：  
  - 在对应模块中做最小必要修改；  
  - 如修复逻辑有代表性，建议在 SDD 中简要记录。

## 开发与测试

- 使用 Go 1.21+。  
- 开发节奏建议“小步实现 + 立即测试”：每完成一个小能力块，尽快补充并运行对应测试。  
- 运行全部测试：

```bash
go test ./... -cover
```

## 提交与 PR

- 单次提交聚焦一个变更点，提交信息格式推荐：`模块: 简要说明`。  
- 在 PR 中简要说明动机、主要改动点和运行过的测试命令。  
- 如改动影响协议、安全或部署，请在描述中明确指出。
